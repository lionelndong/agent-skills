# Patterns — Marketing Machine

Every node and tactic as a reusable, hand-off-able pattern. Each is a checklist you can give a team.

## Lifecycle Ads
**When to use**: you record customer calls (remote/SaaS/coaching/agency).
**How**: Legal adds consent language → IT records all calls + buys storage, turns off auto-delete → name calendar events by client → Sales sends happy calls to marketing weekly → Marketing clips **before** (sales call) + **during** (1–2 support calls) + **after** (upsell/milestone) → friendly labeling system → run them.
**Trade-offs**: highest editing horsepower; install **last**. Most unique journey-style proof.

## "Upon Delivery" Ads (local/physical)
**When to use**: you have a physical delivery moment (build reveal, scale, veneers, "you got the job!").
**How**: capture the **raw reaction** → use it as the **hook (first 3–5 sec)** → make per-reaction ads + reaction-only highlight reels.
**Trade-offs**: only works with a tangible reveal; reels "crush."

## Social Media Scrape
**When to use**: you/customers post; you have a taggable profile.
**How**: create a **hashtag/saying** + profile to tag → reminders to grab tagged media everywhere (esp. 24-hr stories) → save with searchable names → **studs vs. duds** folder → run best media with already-performing copy.
**Trade-offs**: depends on community posting volume. Tell: good organic ≈ good ad/outbound performance.

## Communication & Communities Scrape (chats / community / reviews)
**When to use**: you talk to customers daily (you or reps).
**How**: build a **key-term list** (PIF, Sales, $, Profit, Revenue, close, leads, show, scheduled, win) → shared folder → reminder to search terms, sort by date, screenshot new wins → community "wins" category (Fri prompt, Mon screenshot) + screen-record the wins scroll → reviews: list sites (Glassdoor/Yelp/Google/FB/Trustpilot/Amazon), sort newest, screenshot → **reward the person who turns them in** (optional $100/wk gift card for a screenshot that becomes a winning ad).
**Trade-offs**: nearly free, high volume; dies without manager rewards.

## In-Person Events — Stage Ads
**When to use**: any event you run (do events 1–2×/yr).
**How**: on-stage photo with crowd celebrating behind you + multiple selfies + video; shoot **end of day 1 / before lunch day 2**; one take.
**Trade-offs**: great size/scope proof; single chance to capture.

## In-Person Events — Professional Testimonials
**When to use**: events with pre-selected high-result customers.
**How**: reserve a **recording room** (or 2–3 hotel rooms) → hire ≥1 extra camera team (best team on testimonials) → set lighting/sound, **TEST×3** → confirm customers in writing, give keynote notes → **clipboard hit list** + announced **handler** → incentivize (customer bonus; **+$50/handler** per off-list capture) → run the **6-point script** → nail the **hook**, get the **name first** → save/upload/music/run.
**Trade-offs**: most valuable event proof; fails badly if audio/video is lost.

## In-Person Events — Man in the Street
**When to use**: events with crowds on breaks.
**How**: 2nd two-man team → prep **10 core questions**, get **10+ answers each** → fast-hitting highlight reels per question (or piecemeal the 6-point script into a "mondo testimonial").
**Trade-offs**: seconds to capture; high proof density around one metric.

## Bonus Unlocks
**When to use**: you can make a "big and sexy" giveaway worth charging for.
**How**: attach a price, make it for sale → offer it **free for a testimonial video** → record the pitch where all customers see it → instruct: <5-min video hitting the 6 points to `proof@yourco.com`, give an example, **post in group with stats** (2nd written testimonial), tag manager, include scrape keywords.
**Trade-offs**: small effort to build the asset; yields two proofs per person.

## Award Testimonials (the FOUR-proof pattern)
**When to use**: you have (or can invent) an aspirational customer goal.
**How**: make an award **tied to the community, big & cool** (e.g. sword & shield for "Gym Lords") → announce what they get → to redeem, require: **PROOF 1** group post w/ real numbers + tag owner; **PROOF 2** emailed video; **PROOF 3** unboxing video; **PROOF 4** on-stage presentation → mail engraved, present generic.
**Trade-offs**: spend on the trophy ("don't do a boring thing"); 4 assets per person.

## Competitions
**When to use**: periodically (1–2×/yr — audiences tire).
**How**: set announce + deadline → **crazy award** (e.g. 6 months free) → instructions (results template, attach video, tag manager, email raw file) → share last winner + 6-point script → announce everywhere, **personally follow up top ~50** → ask for **in-the-moment raw** video → cut off, format as ads, run → **30-day reminder** to count conversions; winner = **most conversions over 30 days**.
**Trade-offs**: big influx, low edit; can't repeat often.

## Operating Cadence (Putting It All Together)
**When to use**: once ≥1 node is live.
**How**: weekly **start-of-week checklist** sweeps the always-on nodes (names, social, chats, community, reviews, bonuses, awards, reaction videos); events + competitions deliver a **~12-week wave**.
**Trade-offs**: lapses without an owner; volume → grand slam ads.

## Install Order (effort-to-influx)
**When to use**: starting from zero.
**How**: 1) Competition 2) Chat scrape 3) Community scrape 4) Reviews scrape 5) Social 6) Bonus unlocks 7) Award unlocks 8) Events 9) Lifecycle.
**Trade-offs**: front-loads big-influx/low-edit nodes; defers the editing-heavy ones.
